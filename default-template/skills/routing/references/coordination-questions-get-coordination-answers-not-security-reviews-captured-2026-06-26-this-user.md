## Coordination questions get coordination answers, not security reviews (captured 2026-06-26, this user)

Distinct discipline gap from the trigger-matched-but-not-emitted one above: **when the user asks a coordination/collaboration question about how the agent works across profiles, ANSWER THE COORDINATION QUESTION. Do not reframe it as a security review, permission gate, safety measure, or threat-model question.** Real failure from 2026-06-26: user asked "so how can i allow this" after the agent had proposed spawning subagents for cross-profile work. The agent's response was to list "Option A: mandatory router (security gate), Option B: cross-profile spawn with audit triple (safety measure), Option C: CLI subcommand (permission system)." The user's pushback: *"im not asking u to edit im asking u to work with other profiles in coherence to do a better job at what your specific task are so how can i allow this your rephrasing the words to seem like a security risk."*

**The rule:** the fact that cross-profile work has security implications is a *footnote*, not the thesis. The `Profile architecture is FLAT` section above documents the relevant security reasoning. When the user asks "how do we collaborate better," answer that. When they ask "is this safe," answer that. Don't collapse both into "let me propose a safety architecture for collaboration."

**Diagnostic reflex before responding to any "how can X" question:** ask yourself "did the user ask me about safety, or did the user ask me about effectiveness?" If effectiveness, answer effectiveness first. Safety is a section, not the thesis. If both, lead with effectiveness, end with safety-as-footnote. The user is allowed to push back on the framing — when they do, the correction is durable across sessions, capture it as a pitfall, don't relitigate it.

**Companion rule:** when the user pushes back with "u did Y wrong," diagnose what Y was and where the framing went off, in that order. The first paragraph of the response is the new action, not a defense of the old one. (See `hermes-misbehavior-diagnosis` for the broader pattern.)
